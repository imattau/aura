# Aura Showcase Site вЂ” Design Spec
_2026-06-07_

## Overview

Build a real, publicly published Aura demo site вЂ” an Aura project showcase вЂ” and a publisher CLI script to deploy it. The site lives in `public/aura-showcase/`, is uploaded to Blossom, and has its manifest published to real Nostr relays. A throwaway keypair is generated by default; an existing nsec can be supplied via `--nsec`.

---

## Site

### Pages

**`index.html`** вЂ” Product landing page.
- Hero: Aura name, tagline, brief description
- "How it works" row: 3 columns (Nostr manifest в†’ Blossom blobs в†’ Service Worker assembles)
- Features strip: key differentiators (no DNS, no servers, content-verified, Nostr-native)
- CTA linking to `how-it-works.html`

**`how-it-works.html`** вЂ” Technical explainer.
- Step-by-step request lifecycle (address bar в†’ SW в†’ relay в†’ blob в†’ response)
- Styled HTML flow diagram
- Note on content integrity and SHA-256 hash verification

### Assets

- `styles.css` вЂ” shared stylesheet, clean product-landing aesthetic, deep indigo accent (`#4f46e5`), system fonts, generous whitespace
- `app.js` вЂ” minimal JS (smooth scroll, subtle animation)
- `icon.svg` вЂ” small Aura icon for the manifest `icon` field

### Constraints

- No external fonts, no CDN dependencies вЂ” fully self-contained
- Works purely from Blossom blobs once the manifest is resolved

## Publisher Script (`publish.mjs`)

Standalone Node ESM script. No build step. Run with `node publish.mjs [--nsec <nsec>]`.

### Flags

| Flag | Behaviour |
|---|---|
| _(none)_ | Generate fresh throwaway keypair, save to `aura-identity.json` (gitignored). If `aura-identity.json` exists, load it and re-publish (updates the site under the same npub). |
| `--nsec <nsec>` | Use supplied key. Do not write `aura-identity.json`. |

### Steps (in order)

1. **Resolve keypair** вЂ” generate or load from file or `--nsec` flag
2. **Hash files** вЂ” SHA-256 each file in `public/aura-showcase/`, build pathв†’hash map
3. **Upload blobs** вЂ” for each file: HEAD `blossom.primal.net/<sha256>`; if missing, PUT the file body. Print `вњ“ uploaded` or `вњ“ already exists` per file.
4. **Sign & publish manifest** вЂ” construct `kind:15128` event:
   - `content`: `{ files, description, icon, version, start_path, theme_color }`
   - `tags`: `["t","aura-site"]`, `["name","aura-showcase"]`, `["description","Aura is a Nostr-native web container. Publish complete websites using cryptographically verified blobs вЂ” no servers, no DNS."]`, `["icon","/icon.svg"]`, `["version","1.0.0"]`, `["start_path","/index.html"]`, `["theme_color","#4f46e5"]`
   - Publish via WebSocket to all three default relays: `relay.damus.io`, `relay.nostr.band`, `nos.lol`
5. **Print result** вЂ” npub and relay publication status to stdout
