How a page load works

From the address bar to rendered HTML — no origin server involved.

1

You navigate to a site

The Aura shell receives a navigation to /~npub1…/index.html. The npub in the path identifies the site owner's Nostr public key.

2

Service Worker intercepts

The installed Service Worker catches the FetchEvent before it reaches the network. No request leaves the browser yet.

3

Manifest fetched from relay

The SW opens a WebSocket to one or more Nostr relays and requests the latest kind:15128 event authored by the site's pubkey. This event contains the full path→hash map for the site.

4

Path resolved to hash

The SW looks up /index.html in the manifest's files map and retrieves the corresponding SHA-256 hash.

5

Blob fetched from Blossom

The SW fetches the blob from a Blossom server using the hash as the URL path: https://blossom.primal.net/<sha256>. Blossom servers are pure content-addressed storage — any server holding the hash can serve it.

6

Hash verified

Before handing the blob to the browser, the SW computes its SHA-256 and compares it to the hash from the manifest. If they don't match, the blob is discarded and the fetch fails.

7

Response constructed locally

The SW constructs a synthetic Response object with the correct Content-Type header and returns it to the browser. The page renders normally — the browser never knew there was no server.

Browser navigate /~npub1…/index.html │ ▼ Service Worker intercepts FetchEvent │ ├─▶ Nostr Relay REQ kind:15128 by pubkey │ ◀── EVENT { files: { "/index.html": "abc123…" } } │ ├─▶ Blossom Server GET /abc123… │ ◀── blob bytes │ ├─ verify SHA-256(blob) === "abc123…" ✓ │ └─▶ Browser Response(blob, { Content-Type: text/html })

Content integrity by default

Because every file is identified by its SHA-256 hash in the manifest — and the manifest is signed by the site owner's Nostr key — a visitor cannot be served a tampered file without the hash check failing. There is no TLS certificate to steal, no CDN to compromise, and no origin server to target.
